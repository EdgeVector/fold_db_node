# Q4 2024 Team Retrospective

## What went well
- Shipped 3 major features on time
- Reduced bug count by 40% compared to Q3
- New CI/CD pipeline cut deploy time from 45min to 12min
- Great collaboration between frontend and backend teams

## What could improve
- Code review turnaround time averaging 2 days (target: 1 day)
- Flaky integration tests causing false failures
- Documentation falling behind feature development
- On-call burden uneven — 2 people handling 80% of pages

## Action items for Q1 2025
1. Implement code review SLA: 24hr for small PRs, 48hr for large
2. Dedicate 1 sprint to fixing flaky tests
3. "Docs day" — last Friday of each month
4. Rotate on-call more evenly, create runbooks for common issues
