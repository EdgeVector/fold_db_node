# Test face fixtures

DALL-E 3-generated portraits of two fictional characters ("alice" and
"bob"), each shown in 3 variations. Used by Phase 1 fingerprint
pipeline tests so we have real detectable faces without needing
anyone's actual photos.

## Files

| File | Character | Size | SCRFD confidence |
|---|---|---|---|
| `alice_01.jpg` | alice | ~1.8 MB | 0.881 |
| `alice_02.jpg` | alice | ~1.5 MB | 0.857 |
| `alice_03.jpg` | alice | ~1.6 MB | 0.877 |
| `bob_01.jpg`   | bob   | ~1.8 MB | 0.842 |
| `bob_02.jpg`   | bob   | ~1.6 MB | 0.883 |
| `bob_03.jpg`   | bob   | ~1.7 MB | 0.848 |

All six images:

- Are 1024×1024 RGB, written by DALL-E 3 as PNG data in a `.jpg`
  filename (SCRFD / `image` crate decode via magic bytes, not
  extension, so this is harmless).
- Contain exactly one high-confidence face each (confidence
  ≥ 0.84 with the standard `buffalo_l` / `det_size=(640, 640)`
  SCRFD config).
- Produce 512-dim L2-normalized ArcFace embeddings via
  `insightface` / `w600k_r50` — the same embedding space
  fold_db's internal `native_index::face` pipeline uses.

## Why these are in the repo

The older `face-a.jpg` / `face-b.jpg` / `face-c.jpg` fixtures
(siblings of this directory) don't contain reliably detectable
faces with the current ONNX model / threshold combo — SCRFD
returns 0 matches on all three. These DALL-E fixtures fix that
so the test framework can actually produce Fingerprint / Mention /
Edge records during `ingest_photo` scenarios without silently
writing empty `ExtractionStatus` rows.

## Clustering behavior

Average cosine similarity between ArcFace embeddings in this
dataset, measured with the same insightface pipeline fold_db
uses:

```
alice ↔ alice: +0.578
alice ↔ bob  : +0.249
bob   ↔ bob  : +0.667
```

Same-character similarity is **2–3× higher** than cross-character,
which means the Phase 1 HNSW cache + Persona resolver can cluster
"alice's 3 photos → one Persona" and "bob's 3 photos → another
Persona" cleanly at typical thresholds. That's the property
scenarios like `face-discovery-3node.yaml` depend on when they
assert that a face search returns matches.

## Regenerating

If you need to replace or extend this set, use
`test-framework/scripts/generate-face-fixtures.py`. It requires
`OPENAI_API_KEY` in the environment and generates a deterministic
set from the character descriptions inline in that script.

```bash
OPENAI_API_KEY=sk-... python3 test-framework/scripts/generate-face-fixtures.py
```

DALL-E 3 does not guarantee facial consistency across generations,
so regenerated images may cluster slightly differently. After
regenerating, re-run the similarity check in the script's `main()`
to confirm same-character similarity is still meaningfully higher
than cross-character before committing.
